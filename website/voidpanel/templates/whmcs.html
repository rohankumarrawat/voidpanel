Overview

The official VoidPanel server provisioning plugin allows web hosting providers running WHMCS to automate their server delivery operations. By connecting to VoidPanel’s secure REST API v2, you can instantly turn paid invoices into live hosting configurations.

Our module operates in the background, communicating via encrypted curl calls. It automatically passes account parameters (domains, emails, passwords) and binds system-level configurations dynamically without manual action.

Prerequisites

Before launching the integration, verify you satisfy the following deployment requirements:

Server Port Check

Verify that your WHMCS hosting node can establish outgoing network requests to your VoidPanel server port (defaults to 8080). If you configure SSL/HTTPS on the panel daemon, port 443 must be accessible.

  • An operational installation of WHMCS (v8.0 or newer).
  • Secure SSL or direct HTTP path access to your VoidPanel admin server.
  • An active **API Token** generated from your VoidPanel super-admin dashboard.

Module Installation

WHMCS Server Setup

Once files are uploaded, register your VoidPanel server node as a provisioning destination inside WHMCS Admin:

  1. Log into your WHMCS Admin Panel.
  2. Navigate to System Settings > Servers (or *Setup > Products/Services > Servers* depending on version).
  3. Click Add New Server.

WHMCS Product Setup

Map your WHMCS billable products directly to your custom VoidPanel hosting packages:

  1. Navigate to System Settings > Products/Services.
  2. Create a new product or edit an existing one (e.g. *Shared Hosting* or *VPS* product type).
  3. Go to the Module Settings tab.
  4. Select Voidpanel in the *Module Name* dropdown.
  5. Choose your registered server group or server.
  6. In the Hosting Package text field, input the **exact name** of the package configured in VoidPanel (e.g., default, basic, or business). Note: Package names are case-sensitive and must match exactly.
  7. Choose your preferred automation option (e.g., *Automatically provision product as soon as the first payment is received*).
  8. Click Save Changes.

Required API Token Scopes

To follow security best practices, do not run connections with root keys that possess global scopes. Instead, configure your API Token in the VoidPanel dashboard with only the following active scopes:

accounts.create
accounts.suspend
accounts.unsuspend
accounts.terminate
accounts.change_package
accounts.change_password

Security Recommendation

Revoke unused scopes (like DNS edits or global backup updates) from the token key dedicated to the WHMCS module to isolate server risks.
