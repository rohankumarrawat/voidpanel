# VoidPanel: Technical Documentation & Developer Guide

VoidPanel is a cutting-edge, hybrid web hosting control panel designed for speed, security, and developer flexibility. It bridges the gap between traditional shared hosting and modern cloud-native deployment.

---

## 🏗️ Technical Architecture & Stack

### 1. **Core Backend: Django (Python)**
VoidPanel's logic is built on **Django 5.x**, utilizing its robust ORM for database management and its flexible view system for the dual-portal architecture.
- **Python Version**: 3.10+ (Recommended 3.12+)
- **Database**: SQLite (Default for simplicity) or MySQL (Production ready).
- **Web Server Gateway**: uWSGI (Performance bridge).

### 2. **Frontend Engine**
- **Styling**: Modern, premium CSS (Glassmorphism design language).
- **Icons**: Font Awesome 6 (CDNs integrated globally).
- **Interactivity**: AJAX-driven UI for real-time service management (Terminal, File Manager).

### 3. **Production Stack (Linux)**
- **Nginx**: Edge server handling port-based routing (8080, 8082, 9002, 8092).
- **Systemd**: All services (uWSGI, BIND9, Postfix, Dovecot, vsftpd) are monitored and managed via standard systemd unit files.

- **Multi-PHP Support**: Individual PHP versioning (8.1, 8.2, 8.3) per domain/subdomain.

## 🛠️ Developer Guide: App Structure

VoidPanel is split into two primary Django apps:

### 1. `panel/` (Admin Dashboard)
The "Superuser" interface. Responsible for:
- Managing users, hosting packages, and global server settings.
- Monitoring collective server status and analytics.
- Deploying initial website and system configurations.

### 2. `control/` (User/Client Portal)
The end-user interface for specific domain owners. Responsible for:
- Domain-specific tools: File manager, Database manager, PHP INI editer.
- Real-time Terminal for domain directories.
- Individual Domain Analytics and resource metrics.

### 🏗️ Model Mapping (`control/models.py`)
- `user`: Custom customer data table.
- `domain`: Core tracking of primary websites and mapping to users.
- `subdomainname`: Management of sub-site logic.
- `allemail`: Mailbox credentials and configurations.
- `mernname`/`pythonname`: Process tracking for advanced apps.

## 🚀 Deployment & Installation

### 📋 Prerequisites
- **OS**: Ubuntu 22.04 LTS or 24.04 LTS (Fresh Installation).
- **RAM**: Minimum 2GB (4GB+ Recommended).
- **Public IP**: Static IPv4 required.

### 🛠️ One-Line Installation
```bash
curl -fsSL https://voidpanel.com/install.sh | sudo bash
```

### 📤 Manual Developer Workflow (voidpanel.com)
1. Zip the repository: `zip -r Archive.zip .`
2. Upload `Archive.zip` to: `https://voidpanel.com/op/install/ubuntu/Archive.zip`
3. Upload `install.sh` and `ubuntu.sh` to their respective hosting paths on `voidpanel.com`.
